Prerequisite CLI tools

The following CLI tools are required for running the exercises in this tutorial. Please have them installed and configured before you get started with any of the tutorial chapters.

Tool macOS Fedora

git

https://git-scm.com/download/mac

dnf install git

Docker

https://docs.docker.com/docker-for-mac/install

dnf install docker

Apache Maven

brew install maven

dnf install maven

stern

brew install stern

sudo curl --output /usr/local/bin/stern -L https://github.com/wercker/stern/releases/download/1.6.0/stern_linux_amd64 && sudo chmod +x /usr/local/bin/stern

yq

brew install yq

https://github.com/mikefarah/yq/releases/latest

httpie

brew install httpie

dnf install httpie

siege

brew install siege

dnf install siege

watch

brew install watch

dnf install procps-ng

kubectx and kubens

brew install kubectx

https://github.com/ahmetb/kubectx

Remote Repository

For some parts of the tutorial, you will be pushing and pulling container images to and from a remote image repository. You will need an account with publish rights. As of this writing, knative works best with Docker Hub.

Download Tutorial Sources

Before we start setting up the environment, let’s clone the tutorial sources and set the TUTORIAL_HOME environment variable to point to the root directory of the tutorial:

git clone -b release/0.4 https://github.com/redhat-developer-demos/knative-tutorial

Setup Workshop Environment

Please set the following environment variables before running the exercises:

# This should be your workshop user id like dev001
$WORKSHOP_USER=''
$TUTORIAL_HOME='`pwd`'

OpenShift Project

For this workshop you have been configured with OpenShift project $WORKSHOP_USER-knativetutorial, change to your workshop project:

oc project $WORKSHOP_USER-knativetutorial
